Ma Veille Technologique

Le Cloud Computing

Définition : Le cloud computing (« informatique dans les nuages ») correspond à l'accès via un réseau de télécommunications, à la demande et en libre-service, à des ressources informatiques partagées configurables.

Concrètement, cela permet de déporter le stockage et la puissance de calcul sur des serveurs distants plutôt que sur sa machine locale. C'est aujourd'hui la base de l'infrastructure web moderne (AWS, Azure, Google Cloud).

Mise en Œuvre

Environnements de Dev

Utilisation d'environnements hébergés dans le cloud pour coder sans contrainte matérielle locale.

Outils : AWS Cloud9, GitHub Codespaces.

Collaboration & Stockage

Partage de fichiers et travail collaboratif en temps réel pour assurer la cohérence des versions.

Outils : Google Drive, OneDrive.

Mots clés suivis : Cloud Computing, Serverless, OpenStack, Docker.
Sources principales : LeMagIT, ZDNet, Blog AWS, GitHub Blog.

Bornages

Cloud Public : Ressources accessibles à tous via Internet (Ex: AWS, Azure). Faible coût d'entrée.
Cloud Privé : Serveurs dédiés à une organisation unique. Plus sécurisé mais plus coûteux à maintenir.
Cloud Hybride : Combinaison des deux : données sensibles en privé, puissance de calcul en public.
Leaders du Marché : 1. AWS (34%)
2. Microsoft Azure (21%)
3. Google Cloud (11%)

Retour d'expérience (REX)

Ce qui fonctionne

  • Le suivi hebdomadaire des sources cloud permet de detecter rapidement les tendances IA/infra.
  • Le bornage par mots-clés (cloud, serverless, souverainete, Kubernetes) evite la surcharge d'information.
  • Le format fiche courte (date, source, resume, impact) facilite la restitution.

Axes d'amelioration

  • Ajouter une source institutionnelle recurrente (ANSSI, ENISA, Arcep) pour l'angle reglementaire.
  • Taguer chaque news par categorie (securite, data, IA, souverainete) pour les comparaisons mensuelles.
  • Conserver une synthese mensuelle en PDF comme trace de veille.
